Preguntas y Respuestas

  1. ¿Qué es un documento XML bien formado? Documento que cumple reglas: etiquetas cerradas, anidamiento correcto, atributos entre comillas, respetar mayúsculas/minúsculas y declarar <?xml version="1.0" encoding="utf-8" standalone="yes"?>.
  2. ¿Por qué usar atributos vs elementos? Atributos describen propiedades; elementos contienen datos que se muestran. Se elige según semántica y necesidades de visualización.
  3. ¿Qué aprendí al convertir a JSON? La estructura jerárquica de XML se mapea a objetos/arreglos; los atributos se representan como propiedades especiales (p. ej. @id).
  4. ¿Cómo se aplica CSS a XML? Con la PI <?xml-stylesheet type="text/css" href="archivo.css"?> y creando reglas para los nombres de etiqueta definidos por nosotros.

Conclusiones

  • XML permite definir estructuras de datos personalizadas manteniendo estándares abiertos.
  • La validación de “bien formado” garantiza interoperabilidad básica entre herramientas y navegadores.
  • CSS puede dar legibilidad a XML directamente en el navegador sin transformaciones adicionales.
  • La conversión a JSON facilita el consumo desde JavaScript y APIs web.
